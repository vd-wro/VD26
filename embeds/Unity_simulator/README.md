# Unity Simulator

During the robot's design and build phase, we developed a Unity simulator.
This simulator was utilized to define the robot's core logic, optimize sensor and actuator placement, and refine its overall functionality.

Unity was an ideal choice due to its robust PhysX physics engine (simulating collisions, gravity, and friction), accurate sensor and camera emulation (providing the robot's perspective and sensory input), and C# scripting capabilities, which allowed us to prototype code similar to the real robot.

Testing before physical prototyping is essential for any robot build, and [simulation with Unity](https://unity.com/es/blog/engine-platform/robotics-simulation-is-easy-as-1-2-3) is becoming more and more common for modern robot engineering.
